# BBOXマージ・フィルタ改善 (2026-04-08〜09)

## 前景マスクの改善

### OR マスク
両画像の前景マスクのOR（どちらかに部品があれば比較対象）を使用。
両方とも背景（黒）の領域だけを除外。片方だけ前景の領域は部品の欠けや
余分な突起の差異として検出対象に含める。

### BG CHROMA のAND条件
背景クロマキー判定を `OR`(どちらかが暗い)→`AND`(両方とも暗い)に変更。
片方だけ部品がある領域を背景として消さない。

## 帯状BBOXフィルタ

E+SIFTパスに `bbox_drop_band_aspect` フィルタを追加。
位置ズレ由来の細長いBBOX（例: 214x8, aspect=26.8）を除去。

```
平面異常: 上端の帯2つ (214x8, 288x8) を除去 → 10→8
```

## 背景ギャップ分離マージ

`merge_nearby_bboxes` にdiff_mask参照を追加。
2つのBBOX間のギャップ領域に差分ピクセルが少ない場合（`bridge_fill_thresh=0.15`）、
背景で分離された別部品とみなしマージしない。

### マージ距離の自動計算

`bbox_merge_distance_ratio: 0.03`（短辺の3%）で自動計算。
固定値に依存せず画像サイズに応じて調整される。

| サンプル | 短辺 | auto距離 |
|----------|------|---------|
| 立体 | 1162 | 34px |
| B | 750 | 22px |
| 平面 | 585 | 17px |
| A | 534 | 16px |
| 線対称 | 439 | 13px |

## ensemble_min_area の修正

`min(固定値, ratio計算値)` → `max(固定値, ratio計算値)` に修正。
固定値200が常に勝ちノイズBBOXが通過していた問題を解消。

`ensemble_min_area_ratio: 0.0003` に調整。
- ノイズ(area≤780): 除去
- 耳(area≥3,876): 保持

## 検討して不採用としたアプローチ

### 差分の方向分離（欠損/余剰）
マスターにあって入力にないもの（欠損）と、入力にあってマスターにないもの（余剰）を
分離してBBOXを生成し合成する案。

**結果**: 立体サンプルで欠損14+余剰5と分類されたが、表面テクスチャ差で
方向判定がノイズまみれになり、耳単位での分離には至らなかった。
前景マスクの粒度が粗く、ピクセル輝度差ベースでは精度不足。
